<Note>
**What is Vibe Coding?** It's a new wave of AI-native development where you train your AI assistant with domain-specific context, then collaborate naturally to build complex applications. No more explaining APIs or searching documentation - your AI knows Multisynq inside and out.
</Note>

## How It Works: Two Methods

You can integrate Multisynq with your AI assistant in two primary ways:

<CardGroup cols={2}>
  <Card title="1. Contextual AI Menu" icon="zap">
    **Easiest method.** Use the menu at the top of any page to send the content directly to your favorite AI (ChatGPT, Claude, Gemini). It's a one-click way to give the AI context for a specific task.
  </Card>
  <Card title="2. Full Context Files" icon="files">
    **Most powerful method.** For building entire applications from scratch, provide your AI with a full training file. This gives the AI a deep understanding of the entire Multisynq library.
  </Card>
</CardGroup>

## Method 1: Using the Contextual AI Menu

At the top right of every documentation page, you'll find a menu with several AI-powered options. This is the quickest way to get help with specific questions or code snippets.

<AccordionGroup>
    <Accordion title="What do the options mean?">
        - **Copy Page**: Copies the current page's content as clean Markdown. Useful for pasting into any AI tool as context.
        - **View as Markdown**: Opens a new tab with the raw Markdown of the current page.
        - **Open in ChatGPT/Claude/Gemini**: Starts a new conversation with the respective AI, automatically pre-loading the current page's content as context.
        - **Add Multisynq MCP**: For advanced users, this provides the command to install the Multisynq Model Context Protocol server, connecting AI applications directly to Multisynq's functions.
    </Accordion>
</AccordionGroup>

### How to Use It
<Steps>
  <Step title="Navigate to a Page">
    Find a documentation page relevant to your task (e.g., the `useStateTogether` hook page).
  </Step>
  <Step title="Open the Menu">
    Click the menu icon at the top right of the page.
  </Step>
  <Step title="Choose an Option">
    Select "Open in Claude" (or your preferred AI). A new tab will open with the conversation ready to go.
  </Step>
  <Step title="Ask Your Question">
    The AI now has the context of the page. Ask your question, for example: *"How do I use this hook to sync a user's drawing color on a whiteboard?"*
  </Step>
</Steps>

        ```javascript
        // Three.js + Multisynq Integration Pattern
        import * as THREE from 'three';
        import * as Multisynq from "@multisynq/client";

        class World3DModel extends Multisynq.Model {
            init() {
                this.objects = new Map(); // Map<objectId, {position, rotation, scale}>
                this.users = new Map();   // Map<viewId, {position, rotation}>
                this.subscribe(this.sessionId, "userMove", this.handleUserMove);
            }

            handleUserMove({ viewId, position, rotation }) {
                this.users.set(viewId, { position, rotation });
                this.publish(this.sessionId, "worldUpdate", this.getWorldState());
            }

            getWorldState() {
                return {
                    users: Object.fromEntries(this.users),
                    objects: Object.fromEntries(this.objects)
                };
            }
        }
        World3DModel.register("World3DModel");
        ```

## AI Assistant Tips & Magic Prompts

<CardGroup cols={2}>
  <Card title="ChatGPT/Claude/Gemini" icon="message-circle">
    1. Start a new conversation.
    2. Paste one of the full context files.
    3. Wait for the AI to acknowledge it has read the context.
    4. Add the "Magic Prompt" with your app idea.
  </Card>
  <Card title="GitHub Copilot" icon="code">
    1. Create a new file (`context.md`) and paste the context.
    2. In your main code file, add the Magic Prompt in comments.
    3. Start coding - Copilot will now provide Multisynq-aware suggestions.
  </Card>
</CardGroup>
