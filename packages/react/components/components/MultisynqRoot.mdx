## MultisynqRoot Component

<a id="multisynqroot"></a>
<p>This class extends the Multisynq [Model](../client/Model.html) class to automatically update React state when the model changes. We recommend that your models inherit from this class, as it requires less code, but you can still use the Model class. Keep in mind that if you do, you won't be able to use the <a href="#usereactmodelroot">useReactModelRoot</a> hook, and will have to publish and subscribe to the input and output events on your own. For more details about this, please check the [Event overview](../client/index.html#events) and the [View.publish](../client/View.html#publish), [Model.subscribe](../client/Model.html#subscribe), [Model.publish](../client/Model.html#publish), and [View.subscribe](../client/View.html#subscribe) methods.</p>
### Props
<ResponseField>
<ResponseField name="viewId" type="string">
<p>The id of the view that just connected</p>
</ResponseField>
<ResponseField name="viewInfo" type="ViewInfo">
<p>The ViewInfo object for the joined view</p>
</ResponseField>
<ResponseField name="viewId" type="string">
<p>The id of the view that just left</p>
</ResponseField>
<ResponseField name="viewInfo" type="ViewInfo">
<p>The ViewInfo object for the view that left the session</p>
</ResponseField>
<ResponseField name="{object}" type="any">
<p>{object} sessionParams The session parameter object that is passed to [Session.join](../client/Session.html#.join). If <code>name</code> or <code>password</code> are undefined at the time of joining a session, random values will be generated</p>
</ResponseField>
<ResponseField name="[showChildrenWithoutSession=false]" type="boolean">
<p>If true, children will be rendered even if there's no active session.</p>
</ResponseField>
<ResponseField name="[deferSession=false]" type="boolean">
<p>If true, the session creation will be deferred.</p>
</ResponseField>
</ResponseField>
### Usage
<Tabs>
  <Tab title="Example 1">
  ```javascript
  const multisynqView = useContext(MultisynqContext);
  const model = multisynqView.model;
  /
  let MultisynqContext = {};
  
  /**
  Main wrapper component that manages a Multisynq sessions, enabling child elements to use the hooks described below.
  It takes the same parameters as [Session.join](../client/Session.html#.join) except that it doesn't need a root View class,
  since multisynq-react provides a suitable View class behind the scenes.
  ```
  </Tab>
  <Tab title="Example 2">
  ```javascript
  function CounterApp() {
    return (
      <MultisynqRoot
        sessionParams={{
          name: import.meta.env["VITE_MULTISYNQ_NAME"],
          password: import.meta.env["VITE_MULTISYNQ_PASSWORD"],
          appId: import.meta.env["VITE_MULTISYNQ_APP_ID"],
          apiKey: import.meta.env["VITE_MULTISYNQ_API_KEY"],
          model: CounterModel,
        }}
      >
        <p>Your application here</p>
      </MultisynqRoot>
    );
  }
  ```
